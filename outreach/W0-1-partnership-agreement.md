## 1. 사업 정의

- **사업명**: StoreScope (사이트: ddookim.github.io/storescope/)
- **모델**: 드롭시퍼 + e-commerce 운영자 대상 크로스스토어 트렌딩 인텔리전스 SaaS ($19 Starter / $49 Pro)
- **유입 경로**: SNS 숏폼 + Reddit + 커뮤니티 인바운드 (B2B 콜드메일 없음)
- **인프라**: Cloudflare Quick Tunnel → 60일 내 Render free tier 이전

## 3. 수익 분배

- **Net revenue (결제 - 환불 - Paddle 수수료 - 인프라 비용) 50:50 분배**
- 인프라 비용 = Render paid + 도메인 + 이메일 + 분석 도구 (월 합계, 영수증 보관)
- 정산 주기: 매월 마지막 주 일요일
- 정산 계좌: 양측 별도 통보, 변경 시 30일 사전 고지

## 4. 지적재산권 (IP)

- **공동 소유 100%**: 코드, 데이터셋, 콘텐츠 (SNS 영상, 블로그, Reddit 글), 브랜드, 도메인, 고객 리스트
- 일방 단독 사용 금지. 외부 라이센싱·매각 시 양측 사전 서면 합의 필수
- GitHub repo (ddookim/storescope) collaborator 권한 양측 동일

## 5. Exit 트리거 (자동)

다음 신호 시 **양측 서면 합의 없이도** 자동 발동:

| 신호 | 액션 | 시한 |
|---|---|---|
| D+30 paid < 5 AND 환불률 > 35% | 모델 재정의 회의 (1h sync) | 7일 내 |
| D+60 MRR < $100 | 사업 KILL 또는 핵심 자산 매각 검토 | 14일 내 |
| 일방이 30일 연속 약속 시간의 50% 미만 투입 | 다른 측이 30일 사전 고지 후 단독 운영권 승계 가능 | 명시 |
| 양측 합의로 사업 종료 결정 | Acquire.com / RapidAPI / 데이터셋 매각 3트랙 가동 | 30일 내 |

**Exit 시 자산 buy-out 권**: 사업 종료 결정 후 30일 내 일방이 사업 단독 인수 의사 표명 시, 직전 6개월 평균 MRR × 12 의 50% 를 다른 측에 지급하여 단독 권리 확보 가능 (계산 기준 합의 시점의 MRR 적용).

## 6. 분쟁 해결

- 1차: 양측 1:1 대화 (1주일 내)
- 2차: 양측 합의 제3자 중재 (커뮤니티 내 신뢰 인물)
- 3차: 대한민국 법 기준 서울중앙지방법원
